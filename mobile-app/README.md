# FitWayHub Mobile

App-only build of FitWayHub for iOS and Android, packaged via [Capacitor](https://capacitorjs.com/).

This folder is a self-contained duplicate of the web client with the public
marketing website (`/`, `/about`, `/blogs`, `/privacy`, `/terms`, etc.) stripped
out. Athlete, coach, admin, and auth routes are all included. It talks to the
same backend as the web build — only the frontend is split.

## Setup

```bash
cd mobile-app
npm install
cp env.example .env   # then set VITE_API_BASE to your backend URL
```

## Develop in the browser

```bash
npm run dev
```

The dev server proxies `/api` and `/uploads` to whatever `VITE_API_BASE` points
at (defaults to `http://localhost:3000`).

## Build for native

The first time you want to ship to a device, add the native projects:

```bash
npm run cap:add:android   # generates android/
npm run cap:add:ios       # generates ios/  (macOS only)
```

Then build + sync the web bundle into the native shells:

```bash
npm run build:android     # vite build && cap sync android
npm run build:ios         # vite build && cap sync ios
```

Open the native IDE to install on a device or upload to the stores:

```bash
npm run cap:open:android  # opens Android Studio
npm run cap:open:ios      # opens Xcode
```

Or run straight on a connected device:

```bash
npm run cap:run:android
npm run cap:run:ios
```

## Notes

- `capacitor.config.ts` is shared in spirit with the web repo — same `appId`
  (`com.peetrix.fitwayhub`), same splash + status-bar config.
- `vite.config.ts` automatically sets `base: './'` when `CAPACITOR_BUILD=true`,
  which is what `cap sync` expects. The `npm run build:*` scripts set this for
  you.
- The mobile build hits the production API by way of `VITE_API_BASE` (set in
  `.env`). Keep this pointed at the deployed backend; the `server/` folder
  isn't copied here.
- `android/` and `ios/` are gitignored — they're generated by `cap add` and
  carry no source you can't regenerate. Commit them only if you need to keep
  per-project native config (Gradle tweaks, Info.plist edits, signing).
